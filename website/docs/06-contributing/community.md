connectCenter (formerly **Score**) is an open-source project stewarded by the
[Open Applications Group (OAGi)](https://oagi.org) in collaboration with the
[National Institute of Standards and Technology (NIST)](https://www.nist.gov/services-resources/software/score-standards-life-cycle-management-tool). This
page explains where to get help, where discussion happens, and how to stay in
the loop.

## Project home

The canonical source repository lives on GitHub:

- **Source & releases:** [github.com/OAGi/Score](https://github.com/OAGi/Score)
- **Organization:** [Open Applications Group (OAGi)](https://oagi.org)

connectCenter is distributed under the **MIT License** (see `LICENSE.txt` in the
repository root).

## Getting help & discussion

Most project conversation happens in the open on GitHub. If you have a question,
a bug, or a feature idea, start there:

- **Ask a question or report a bug** — open a
  [GitHub issue](https://github.com/OAGi/Score/issues). Search existing issues
  first to avoid duplicates; the maintainers and community use issues both for
  tracking work and for clarifying discussion.
- **Propose or discuss a change** — open a pull request against the repository,
  or comment on the related issue. See [Issue Management](./issue-management.md)
  for how issues are triaged and labeled, and
  [Development Setup](./development-setup.md) for building the project locally.

For broader CCS / CCTS (Core Component Specification) standards questions that go
beyond the tooling itself, the [OAGi community](https://oagi.org) is the right
audience.

## How to be a good community member

- **Search before you post.** Check open and closed
  [issues](https://github.com/OAGi/Score/issues) for an existing answer.
- **Be specific.** Include the connectCenter version (for example **3.5.2**), how
  you deployed it, and the exact steps to reproduce. See the
  [Architecture Overview](./architecture.md)
  for module names and the [Deployment guide](../02-getting-started/deployment.md)
  for the standard Docker Compose setup.
- **Stay on topic.** Use GitHub issues for the tooling; use OAGi channels for
  standards-level questions.
