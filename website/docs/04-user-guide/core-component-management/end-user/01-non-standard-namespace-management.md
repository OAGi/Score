---
title: "Non-standard Namespace Management"
sidebar_position: 1
---

All EUCCs require a Non-standard namespace (i.e., end user namespace) before they can leave the WIP state or pass release validation.
Standard namespaces (i.e., developer namespaces) cannot be used in an EUCC; the "Namespace" drop-down of an EUCC offers Non-standard namespaces only.

An end user can [create](../developer/02-namespace-management.md#create-a-namespace) and [edit](../developer/02-namespace-management.md#edit-detail-of-a-namespace) a namespace he/she owns.
A namespace created by an end user is always a Non-standard namespace: the "Standard" checkbox on the "Create Namespace" page is unchecked and locked.

![Create Namespace page as an end user with the URI filled out and the Standard checkbox unchecked and locked](/img/user-guide/eu_namespace_create.png)

He/she can also [find](../developer/02-namespace-management.md#find-a-namespace) and [view](../developer/02-namespace-management.md#view-detail-of-a-namespace) the details of a namespace.
Finally, he/she can [discard](../developer/02-namespace-management.md#discard-a-namespace) an owned namespace that is not in use, or [transfer the ownership](../developer/02-namespace-management.md#transfer-ownership-of-a-namespace) of an owned namespace to another user.
It should be noted that the transfer of the ownership of a Non-standard namespace can occur only between end users.
